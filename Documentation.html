In addition to the solver, a parser is provided which translates bilevel problems specified by an .mps and a .aux file to the scalars, vectors and matrices required by the solver.

Furthermore, code for analyzing the data from numerical tests in the form of performance profiles and means and medians of runtimes and other characteristics is provided.

Dependencies

The Solver is based on Gurobi 9.1 and thus requires a Gurobi license and the installation of Gurobipy. Furthermore, the solver requires the installation of Numpy and Scipy. To use the file reader that parses MILP-LPs from an .mps and a .aux file, the installation of Pysmps is required. For creating performance profiles, Pandas and Matplotlib are required.

Usage

MIQP-QPs can be solved by the two benchmark approaches used by Kleinert, Grimm and Schmidt. This is provided by the function optimize_benchmark in the file Solvers/benchmarks.py. The desired approach can be chosen by setting the approach-parameter.

approach Description
KKT-MIQP The KKT-based MIQP reformulation (2) in the paper (p. 464) is solved by Gurobi.
SD-MIQCQP The strong-duality based convex MIQCQP reformulation (15) in the paper (p. 468) is solved by Gurobi.

Both the results from the authors and the Bachelor thesis indicate that SD-MIQCQP is the faster and more reliable approach.

MIQP-QPs can also be solved by single-tree and multi-tree based approaches proposed by Kleinert, Grimm and Schmidt. These algorithms are provided by the function solve in the file Solvers/multitree.py. It solves MIQP-QPs with a desired configuration from the paper. The user can choose how subproblems should be solved by setting the subproblem_mode-parameter accordingly.

subproblem_mode Description
regular The single-level reformulation (10) from the paper (p. 467) which is nonconvex but convex with fixed xIx_IxI​ is solved.
remark_1 The subproblem solution is obtained according to Remark 1 in the paper (p. 479). This mode is not recommended.
remark_2 The subproblem solution is obtained according to Remark 2 in the paper (p. 480). Can only be used if GlG_lGl​ is strictly positive definite.

The authors found that remark_2 results the shortest running times. remark_1 on the other hand is not significantly better than regular. Additionally, remark_1 caused numerical instabilities in the implementation. Thus, it is recommended to use remark_2 if GlG_lGl​ is strictly positive definite or else regular.

The user can also choose if a multi-tree based or a single-tree based solution approach is used and specify the configuration by setting the algorithm-parameter.

algorithm Description
MT Basic Multi-tree approach.
MT-K Like MT with additional Kelley-type cutting planes.
MT-K-F Like MT-K but the master problem is terminated with the first improving integer solution.
MT-K-F-W Like MT-K-F but the master problem is warmstarted with the current best solution.
ST Basic Single-tree approach.
ST-K Like ST with additional Kelley-type cutting planes.
ST-K-C Like ST-K but a bilevel feasible solution is used to derive an initial strong-duality cut.
ST-K-C-S Like ST-K-C but the initial master problem is warmstarted with a bilevel feasible solution.

In the numerical study of the paper it was found that ST-K-C-S ist the best performing algorithm. The results of the bachelor thesis however indicate that ST is fastest configuration for the Python-solver on small problem instances. In the following, the full documentation of the solve function is provided.

The numerical study is based on problem instances from MIPLIB. The instance specifics are provided by a .mps file which specifies the variables and constraints and a .aux file which assigns the variables and constraints to the upper-level problem, respectively the lower-level problem. The function mps_aux_reader in the file Solver/file_reader.py converts these two files to the vectors and matrices required for the MIQP-QP solver. In addition, the parser adjusts the problem so that it is in line with the MIQP-QPs studied by Kleinert, Grimm and Schmidt, i.e. it makes all lower-level variables continuous and all linking variables integer if not already so. Note that the resulting problem is an MILP-LP. The quadratic terms are generated randomly.

Performing a numerical study

In the file performance_check.py test instances were generated and a numerical study was performed. The parse function mps_aux_reader was used to read in MILP-LP instances. The function quadraticTerms was used to generate random matrices for the quadratic terms to generate MIQP-QP instances. Each instance was saved as a .npz-file. The function run_tests_final was then used to to solve each problem instance with the solution approaches that should be examined. The results (solution status, running time, objective value, number of solved subproblems, time spent in subproblems) were stored in a .txt-file.

To test the optimized binary expansion, test instances with nonzero lower-bounds upon the integer variables were required. The function shift_x was used to shift the feasible set and objective function of a problem into the direction of integer variables while keeping the geometrical properties of the problem instance. Problem instances with integer lower bounds of 1,3,7,15,…, 511,1023 were generated. This unfortunately only rendered stable results for three instances (based on the MIPLIB instances stein27-0.100000, enigma-0.100000, stein27-0.900000). To get a larger test set, different MIQP-QPs were generated from these three instances by using different seeds for the random quadratic terms. In the function test_optimized_binary_expansion all these different instances were solved with and without the optimized binary expansion and the results were stored in a .txt-file too.

In the file performance_profile.py the results of the numerical study were analyzed. The function get_test_datawas used to create a Pandas data frame from the .txt-file. The function plot_performance_profile can then be used to plot performance profiles of a desired subset of solution approaches. The function mean_median_df can be used to create a Pandas data frame of mean and median running times, numbers of solved subproblems and times spent for solving subproblems of a desired subset of solution approaches on the instances which all approaches could solve.

To analyze the effectiveness of the optimized binary expansion, the function opt_bin_exp_plot was used to create a plot which depicts the mean and median running times and midspread of running times for the both the solver with and the solver without the optimized binary expansion for a given distance of the integer lower bound from the origin. The function bin_exp_perf_profile was used to plot performance profile of both the solver with and the solver without the optimized binary expansion for a given shift away from the origin.
